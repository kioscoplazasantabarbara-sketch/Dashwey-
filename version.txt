═══════════════════════════════════════
CAMBIOS v1.2.9-dev
═══════════════════════════════════════

1. FUENTE DE VERDAD: version.json
   - version.json es el único lugar donde se define la versión
   - Todos los demás archivos derivan de él

2. PROPAGACIÓN AUTOMÁTICA via bump.sh
   - Un solo comando: ./bump.sh
   - Incrementa PATCH automáticamente (X.Y.Z → X.Y.Z+1)
   - Propaga a: index.html, sw.js, version.txt, version.json
   - Genera ZIP con nombre correcto: dashwey-{version}.zip

3. PUNTOS UNIFICADOS
   - <title> en index.html
   - manifest/icon cache-bust en index.html
   - _APP_VERSION constante JS
   - CACHE_NAME en sw.js
   - version strings en sw.js push payload
   - version.txt primera línea

4. PROHIBIDO (eliminado)
   - Nombres tipo updatev109, v107
   - Versiones hardcodeadas desincronizadas
   - ZIP con nombre diferente al version.json

═══════════════════════════════════════
ARCHIVOS ENTREGADOS
═══════════════════════════════════════

  index.html      ← v1.2.9-dev + What's New actualizado
  sw.js           ← CACHE_NAME: dashwey-v1-2-9-dev

  version.txt     ← este archivo
  bump.sh         ← script de build unificado
