DIAGNÓSTICO: 3 bugs estructurales (no uno)
═══════════════════════════════════════════════════════════

BUG A — Sheet DOM con onclicks hardcodeados a App.dash.*
  El overlay (#dash-period-sheet) tenía:
    onclick="App.dash.closeGlobalPeriodPicker()"  ← siempre Dashboard
    onclick="App.dash.applyGlobalCustom()"         ← siempre Dashboard
  Cuando Proveedores (u otro módulo) abría el sheet:
  · Tocar el fondo cerraba vía App.dash.* ignorando el config del módulo
  · Apply llamaba applyGlobalCustom() en vez de config.onCustom
  openPeriodSheet() sobreescribía applyBtn.onclick ✅ pero el overlay
  es atributo HTML estático — JS nunca lo tocaba → siempre App.dash.*

BUG B — 'personalizado' en Proveedores esperaba DOM que no existe
  OPTS_PROV ofrecía la opción 'personalizado' que en saveProvEdit
  intentaba leer pe-cada-dias / pe-fecha-ini / pe-fecha-fin.
  Esos IDs no existen en ningún HTML del sheet → siempre cadaDias:''
  La opción era inutilizable silenciosamente.

BUG C — closeGlobalPeriodPicker no limpiaba _periodConfig
  Al cerrar el sheet por overlay, App.dash.closeGlobalPeriodPicker()
  solo quitaba .open pero dejaba sheet._periodConfig con el config
  del contexto anterior → la siguiente apertura heredaba config stale.

═══════════════════════════════════════════════════════════
SOLUCIÓN: UI compartida, lógica separada por módulo
═══════════════════════════════════════════════════════════

FIX A — Desacoplar el DOM del sheet de App.dash.*
  Nuevas funciones en Utils (módulo neutral):
    Utils._onPeriodSheetClose() — lee sheet._periodConfig, cierra limpiamente
    Utils._onPeriodSheetApply() — lee config.onCustom, delega correctamente
  DOM del sheet ahora apunta a estas funciones genéricas:
    onclick="Utils._onPeriodSheetClose()"  (overlay)
    onclick="Utils._onPeriodSheetApply()"  (botón Apply)
  openPeriodSheet() ya no sobreescribe onclick del botón Apply
  (lo hace la función Utils en tiempo real via _periodConfig)

FIX B — Eliminar opción sin UI de OPTS_PROV
  'personalizado' eliminado de OPTS_PROV — requería campos DOM inexistentes
  5 tipos operativos: diario · semanal · quincenal · mensual · custom(rango)
  saveProvEdit limpiado — ya no intenta leer pe-cada-dias ni pe-fecha-*
  La periodicidad se guarda en tiempo real vía _refreshProvPeriodBtn

FIX C — closeGlobalPeriodPicker limpia _periodConfig
  Añadido sheet._periodConfig = null en closeGlobalPeriodPicker()
  Garantiza que cada apertura del sheet empiece desde estado limpio
  sin importar qué módulo cerró el sheet anteriormente
