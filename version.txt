ARQUITECTURA: Utils.openPeriodSheet
═══════════════════════════════════════════

Problema anterior:
  3 implementaciones distintas del mismo patrón picker de fechas/periodo:
  1. openGlobalPeriodPicker → #dash-period-sheet (HTML + CSS dps-*)
  2. _openGastosPeriodPicker → modal dinámico con ~80 líneas de inline styles
  3. _openVisitasPeriodPicker → modal dinámico con ~60 líneas de inline styles
  + nuevo selector en openEditProv (v9.5.91 parcial)

Solución: Utils.openPeriodSheet(config)
  · Componente único que reutiliza el #dash-period-sheet existente
  · Configurable: título, opciones, opción activa, callbacks
  · Soporta: selección rápida (presets) + rango personalizado (fechas)
  · Accesible desde cualquier módulo: Utils.openPeriodSheet({...})
  · Sin duplicidad de HTML, CSS ni lógica

API pública:
  Utils.openPeriodSheet({
    title,          // Título del sheet (ej: "Periodo de gastos")
    opts,           // Array de { t, i, l, s } — tipo, icono, label, sublabel
    current,        // Opción actualmente seleccionada
    customDesde,    // Valor inicial campo "Desde" (datetime-local)
    customHasta,    // Valor inicial campo "Hasta"
    customLabel,    // Texto del botón "Aplicar" en modo custom
    onSelect(t),    // Callback al seleccionar una opción predefinida
    onCustom(d,h),  // Callback al aplicar rango personalizado
  })

  Utils._onPeriodSheetSel(t)  // Llamado desde onclick de dps-opt

[1] openGlobalPeriodPicker (Dashboard)
    Antes: generaba las opciones inline y abría el sheet directamente

    Después: usa Utils.openPeriodSheet con opciones propias

    es no-op (compatible)

[3] _openVisitasPeriodPicker (Dashboard/Agenda)
    Antes: creaba modal dinámico + lógica de custom form inline
    Después: usa Utils.openPeriodSheet con 7 opciones
    _applyVisitasPeriodCustom ahora acepta params directos (desde, hasta)
    visitas-period-modal ya no se crea — .remove() es no-op

[4] Selector periodicidad en openEditProv (ALM)
    Antes: grid de 4 chips inline + campos de fecha dentro del formulario
    Después: botón .pe-period-btn → abre Utils.openPeriodSheet
    _openProvPeriodSheet: 6 opciones específicas de proveedor
