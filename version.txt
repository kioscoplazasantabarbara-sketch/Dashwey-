Dashwey v9.5.64 — DashweyBus SSOT + Cascade Delete

ARQUITECTURA AÑADIDA — DashweyBus (event bus tipado)

La arquitectura de Dashwey ya era una SSOT (_state único, State.get/set como
únicos accesores). Lo que faltaban eran las conexiones entre mutaciones y renders.
v9.5.64 completa esas conexiones sin reescribir lo que ya funcionaba.

[1] DashweyBus — event bus con batching 0ms
    · window._DashweyBus disponible para debug
    · Deduplica eventos del mismo tipo en el mismo tick
    · 10 event types: venta, pedido, stock, cuenta, gasto, ingreso, visita,
      proveedor, periodo, full
    · Cada handler activa SOLO los renders necesarios (no full dash.render())

[2] Wiring de todos los puntos de mutación → DashweyBus.emit()
    · addVenta     → emit('venta')   → dash.render(), alm.renderOrden
    · addPedido    → emit('pedido')  → dash.renderFinanciero, renderAgenda, alm
    · updateProd   → emit('stock')   → dash.render (kpis)
    · borrarProv   → emit('proveedor') → dash.renderAgenda, alm.updateProvBar
    · addCuenta    → emit('cuenta')  → dash.renderFinanciero
    · addGasto     → emit('gasto')   → dash.renderFinanciero
    · addIngreso   → emit('ingreso') → dash.renderFinanciero
    · addVisita    → emit('visita')  → dash.renderAgenda

[3] State.get.ingresosFin() — getter añadido (faltaba)
    · State.set.addIngreso() + setIngresos() añadidos
    · _confirmarIngreso() ahora loguea en ingresosFin (income log)
    · insights de ingresos-fin ya no fallan silenciosamente

[4] borrarProv cascade delete
    · Elimina el proveedor de la lista
    · Limpia ordenItems del proveedor eliminado
    · Marca productos asociados con prov:null (no los elimina — preserva historial)
    · Emite 'proveedor' → agenda + dashboard se actualizan

[5] dash return object extendido
    · renderCuentas, renderGastos, renderIngresosFin expuestos
    · Permite que DashweyBus llame renders selectivos sin full render
