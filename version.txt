DIAGNÓSTICO FORENSE — CAUSA RAÍZ
═══════════════════════════════════════════

Síntoma: al pulsar "Aceptar pedido" el botón desaparece sin ejecutar nada.
Al hacer scroll el botón reaparece. Al pulsarlo de nuevo → mismo resultado.

Bug #1 (PRINCIPAL) — _guardarCarritoYPopup:
  ANTES:
    acceptBtn.style.display = 'none';       // ← OCULTA EL BOTÓN PRIMERO
    if (!Object.keys(_ordenCarrito).length) return; // ← valida después
    _openAceptarPopupFromCarrito(sel);

  El botón se ocultaba ANTES de verificar el carrito y ANTES de abrir el popup.
  Cualquier fallo silencioso en _openAceptarPopupFromCarrito dejaba el UI con:
    · Botón oculto (display:none puesto)
    · Popup que nunca apareció
    · Pedido sin procesar
  El scroll llamaba _updateAcceptBtn → botón reaparecía → ciclo sin fin.

Bug #2 — _onAcceptBtnTap:
  · Haptic era 'error' (vibración de error al hacer un pedido correcto)
  · Guard de doble-tap (processing) se liberaba con setTimeout(140ms)
    independientemente de si la acción completó o falló
  · La acción (_guardarCarritoYPopup) corría sincrónicamente (0ms)
    mientras el guard se liberaba en 140ms → ventana de doble-tap

Bug #3 — _cancelarPedidoCarrito:
  · Al cancelar el popup, btn.dataset.processing quedaba en '1' para siempre
  · Si el usuario reabría el carrito sin recargar, el botón no respondía

Bug #4 — _confirmarPedidoDesdeCarrito:
  · btn.dataset.processing no se liberaba en el bloque finally
  · Si el finally se alcanzaba por excepción, el botón quedaba bloqueado
