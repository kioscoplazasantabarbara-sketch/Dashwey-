[BUG CRÍTICO — SWIPE CATÁLOGO ALM: CAUSA RAÍZ Y FIX]

═══════════════════════════════════════════
DIAGNÓSTICO FORENSE
═══════════════════════════════════════════

Síntoma reportado:
  · Swipe arriba/abajo sobre un artículo del catálogo no funciona correctamente
  · Swipe horizontal para volver al Dashboard falla desde cualquier artículo
  · Funciona correctamente si el swipe empieza fuera de un artículo

Causa raíz identificada — L12179 (v9.5.77 a v9.5.81):

  function _catInlineLpStart(prodId, el, evt) {
    // v9.5.77: Solo stopPropagation para evitar que el swipe-track capture el gesto
    if (evt) evt.stopPropagation && evt.stopPropagation(); // ← BUG
    ...
  }

Cadena de propagación DOM:
  alm-cat-item [touchstart]
    → alm-cat-inline
      → alm-scroll
        → #tab-alm
          → #swipe-track [touchstart listener — inicializa _sx, _sy]

El stopPropagation() cortaba el bubbling en el ítem.
El swipe-track nunca recibía el touchstart → nunca inicializaba _sx/_sy.
Sin _sx/_sy → el track no podía calcular dx/dy → el swipe horizontal era imposible.

El comentario era incorrecto: el track NO necesita ser bloqueado en touchstart.
El conflicto tap-vs-swipe se resuelve correctamente en touchend (fix v9.5.77, dx>12px).

═══════════════════════════════════════════
FIXES APLICADOS (3 cambios quirúrgicos)
═══════════════════════════════════════════

[1] ELIMINADO stopPropagation() de _catInlineLpStart
    · El touchstart ahora burbujea correctamente hasta el swipe-track
    · El track recibe el evento → inicializa _sx, _sy, _touchStartTime
    · El swipe horizontal al Dashboard funciona desde cualquier artículo
    · El scroll vertical funciona (touch-action:pan-y en .alm-cat-item, v9.5.79)
    · La protección tap-vs-swipe sigue activa en _catInlineLpEnd (dx/dy > 12px)

[2] AÑADIDA función _catInlineLpMove(evt)
    · Cancela el timer de longpress si el usuario se mueve >8px durante el toque
    · Sin esto: un swipe lento (>480ms) disparaba el menú contextual
      aunque el usuario intentara navegar al Dashboard
    · Umbral 8px (más sensible que el 12px del touchend — detección más temprana)
    · Exportada en el return del módulo alm como App.alm._catInlineLpMove

[3] AÑADIDO ontouchmove al HTML de cada alm-cat-item
    · ' ontouchmove="App.alm._catInlineLpMove(event)"'
    · Conecta el movimiento del dedo con la cancelación del longpress

Módulos afectados:
  · _catInlineLpStart — stopPropagation eliminado
  · _catInlineLpMove  — función nueva añadida
  · _buildCatInlineItem — ontouchmove añadido al HTML del ítem
  · return {} del módulo alm — _catInlineLpMove exportada

Comportamiento PRESERVADO sin cambios:
  · Botones stepper (+/-): stopPropagation en sus touchstart/touchend → correcto
    (los botones son hijos del ítem — su stopPropagation es independiente)
  · _catInlineLpEnd: umbral dx/dy > 12px para cancelar tap → intacto
  · touch-action:pan-y en .alm-cat-item → intacto (scroll vertical libre)
  · touch-action:pan-y en .alm-scroll → intacto
  · Longpress 480ms → menú contextual → intacto
