CHANGELOG v1.2.31-dev — Modal central de comprobación de pedido
================================================================

[FEAT] Check-Modal — sistema aislado de recepción de pedidos

ARQUITECTURA
  - window.checkModalState  → estado privado (items snapshot + Set checked)
  - window._checkModal      → API pública: open() / close() / toggle() / confirm()
  - Estado temporal: NO modifica State hasta confirmación final
  - Snapshot inmutable heredado de _pedidoPendiente (ya existente en ALM)

CSS — #check-modal-overlay / #check-modal-box
  - Modal central (no sheet) con backdrop blur(10px) + rgba(0,0,0,0.72)
  - Animación entrada: scale(0.92→1) + opacity(0→1) en var(--sheet-duration)
  - Solo transform + opacity — GPU friendly, cero layout
  - Dark mode completo via tokens del sistema
  - .cm-item: transition opacity sin re-render del nodo
  - .cm-chk: checkbox grande con tick SVG, transición color
  - .cm-price-badge.cm-price-alert: badge rojo con % subida si precio > histórico
  - .cm-confirm-btn: disabled hasta todos marcados, activo con color var(--red)
  - .cm-progress-bar-fill: barra animada con will-change:width

FLUJO DE APERTURA
  Popup análisis → botón "📦 Comprobar recepción"
    → App.alm._abrirCheckModal()
    → cierra aceptar-popup-overlay
    → construye items desde _pedidoPendiente (snapshot ya inmutable)
    → window._checkModal.open(items, {sel, prov})
    → anima entrada (doble rAF)

MARCAR ITEMS (sin re-render global)
  toggle(prodId):
    1. añade/quita class .cm-checked inmediatamente (visual instantáneo)
    2. setTimeout 120ms → appendChild al final del contenedor (reordenamiento)
    3. _updateFooter() actualiza counter + barra + estado del botón

ALERTA DE PRECIO
  - Compara item.precio vs item.precioHistorico (penúltimo registro historialPrecios)
  - Si precio actual > histórico: badge rojo + ⚠️ + % subida
  - Solo visual — no bloquea el flujo

CONFIRMACIÓN (sincronización real)
  confirm():
    1. State.set.updateProd(prodId, {stockActual}) para cada item
    2. State.set.addPedido(pedido) con snapshot completo
    3. App.alm._cancelarPedidoCarrito() — limpia carrito ALM
    4. DashweyBus.emit('pedido_stock') — dash.render() + alm.updateBadge()
    5. document.dispatchEvent('order:checked') + 'data:updated'
    6. Utils.triggerHaptic('medium') + showToast
    7. _checkModal.close()
