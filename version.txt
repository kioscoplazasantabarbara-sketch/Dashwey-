[1A] BUG Header — 2 líneas edge-to-edge visibles
     Causa raíz:
     · #header tenía border-bottom: 1px solid rgba(220,20,60,0.08)
     · Y además box-shadow: 0 1px 0 rgba(220,20,60,0.06) — segunda línea separada
     · Ambas pintaban al mismo tiempo: línea semitransparente + sombra visible
     Fix: eliminado box-shadow, mantenido solo border-bottom con color neutro rgba(0,0,0,0.06)

[1B] BUG Header — Icono de ajustes (engranaje/avatar)
     Causa raíz: el botón mostraba un avatar circular con inicial o foto de perfil
     Fix: reemplazado por hamburger menu — SVG de 3 barras horizontales (22×22px)
     · Rectángulos en y=5, y=10, y=15 con fill:var(--ink) — escala con el tema
     · border-radius:10px en el botón, background:none — integrado visualmente
     · onclick sigue abriendo App.ui.openSettings() — funcionalidad intacta

[1C] JS Avatar — desactivado para no corromper el hamburger
     El JS de _loadProfilePhoto intentaba insertar foto/inicial en el botón
     Fix: código de avatar desactivado — el hamburguer es fijo e inmutable

[2A] BUG Buscador — cuadrado dentro del cajón al pulsar
     Causa raíz: input type="search" en Android Chrome activa un borde/outline
     nativo del sistema + muestra el botón de limpiar propio del browser
     Fix: type="search" → type="text" + -webkit-appearance:none + appearance:none
     en el input #alm-perm-search

[2B] BUG Buscador — X duplicada
     Causa raíz: el browser renderizaba su propio botón ::-webkit-search-cancel-button
     simultáneamente con el botón personalizado #alm-perm-clear
     Fix CSS global:
     .sac-input::-webkit-search-cancel-button,
     .sac-input::-webkit-search-decoration,
     input[type="search"]::-webkit-search-cancel-button { display:none }

[2C] BUG Buscador — "Añadir X" aparece encima del botón "Añadir artículo"
     Causa raíz: al buscar, renderCatInline() añadía inline una fila
     "Añadir X como nuevo artículo" al final del catálogo, Y a la vez
     _permSearch() añadía otra en el dropdown de sugerencias → duplicado visual
     Fix: eliminada la entrada inline de renderCatInline — solo queda en el dropdown

MÓDULO ALMACÉN
══════════════════════════════════════════

[3] BUG Selector cantidad — número duplicado (1 + badge con número)
    Causa raíz: _buildCatInlineItem generaba cuando nc > 0:
    · <div class="alm-cat-badge">nc</div> — número en pill rojo encima
    · <span class="alm-cat-qty">nc</span> — número dentro del stepper
    Ambos mostraban el mismo valor → "1" + "1" en la misma celda
    Fix: eliminado alm-cat-badge de _buildCatInlineItem y _catInlineRefreshItem
    Queda solo el stepper (− N +) que ya muestra el número claramente

[4] BUG Scroll pedido — "Aceptar pedido X bultos" visible al scroll extremo
    Causa raíz: #alm-accept-fixed no tenía position:fixed en el CSS
    El JS hacía display:block + bottom:Xpx via inline style, pero sin
    position:fixed el elemento quedaba in-flow y aparecía al final del scroll
    Fix: añadido CSS completo para #alm-accept-fixed y .alm-accept-btn:
    · position:fixed; left:12px; right:12px; z-index:151
    · El JS sigue calculando bottom = offsetHeight(bottom-tabs) — correcto
