Dashwey v9.5.79

[BUG FIXES CRÍTICOS — GESTOS, CHART Y TIMELINE]

[1] BUG CRÍTICO — ALM: swipe horizontal bloqueado a nivel de browser
    Causa raíz:
    · .alm-cat-item y .alm-scroll tenían touch-action:manipulation
    · "manipulation" = solo permite tap + zoom + pan VERTICAL
    · El browser descartaba el swipe horizontal ANTES de que llegara al swipe-track
    · El track nunca recibía el gesto → transición al Dashboard incompleta/parcial
    Fix:
    · touch-action:manipulation → touch-action:pan-y en .alm-cat-item
    · touch-action:pan-y añadido a .alm-scroll
    · pan-y = scroll vertical libre + propaga swipe horizontal al elemento padre (track)
    · El track del nav recibe el gesto completo → transición correcta a Dashboard

[2] BUG CRÍTICO — Dashboard: barras gráfica X/Y no aparecen
    Causa raíz (triple):
    A) contain:layout en #ev-chart-bars (añadido en v9.5.77):
       · Aísla el elemento del layout de su padre flex (ev-chart-inner)
       · El padre no puede dimensionar el hijo → barras colapsan a height:0
    B) align-items:stretch en ev-chart-inner con contain:layout activo:
       · El intento de stretch no puede atravesar el contain boundary
       · Las barras internas no reciben dimensión del contenedor
    C) Animación con scaleY(0→scale):
       · min-height:3px no se aplicaba porque scaleY(0) colapsa el elemento
       · Con height:0, min-height tampoco funciona → barra invisible
    Fixes:
    A) Eliminado #ev-chart-bars de la lista contain:layout
    B) ev-chart-inner: align-items:stretch → align-items:flex-end
       height:150px fijo en el contenedor como referencia base
    C) ev-bar-col: scaleY(0→scale) → height:0→Xpx
       · hPx = Math.max(3, Math.round((val/yMax) * 150))
       · align-self:flex-end ancla cada barra al fondo del chart
       · Transición: height 0.55s cubic-bezier(0.4,0,0.2,1)
       · ev-chart-x margin-left ya corregido (42px = 36px Y + 6px gap)

[3] BUG — Dashboard: iconos camión/persona no siguen la línea de progreso
    Causa raíz:
    · .tl-track-icon usaba transform:translateX(calc(var(--icon-pct, 4%) - 8px))
    · En CSS, translateX(%) es relativo al WIDTH DEL PROPIO ELEMENTO (16px)
    · translateX(98% - 8px) = translateX(15.68px - 8px) = 7.68px — casi 0
    · El icono siempre quedaba pegado al inicio del track, sin moverse
    Fix:
    · CSS cambiado a: left:var(--icon-pct, 4%) + transform:translateX(-50%)
    · left:% es relativo al ELEMENTO PADRE (.tl-track-wrap) → posición correcta
    · translateX(-50%) centra el icono de 16px sobre el punto exacto del track
    · Transición: left 1.3s cubic-bezier(.25,.1,.25,1) — suave y natural
    · El JS no cambia: sigue escribiendo --icon-pct como custom property
