Dashwey v9.5.59 — Anti-flicker crítico

CAUSA RAÍZ IDENTIFICADA Y CORREGIDA (6 problemas):

[1] _shimmerAllCards() eliminado de render()
    · render() se llama en cada venta/pedido/cambio
    · Shimmer en cada render = 8 appendChild/removeChild por operación
    · Ahora shimmer SOLO se activa en pull-to-refresh (user-triggered)

[2] .card-shimmer background-position → ::before + opacity
    · background-position infinite = paint cada frame = flicker continuo
    · Reemplazado por .is-shimmering::before con animación opacity
    · opacity es compositor-only: cero paint, cero layout

[3] dataHighlight background → opacity pulse
    · CSS background animation = paint en todas las tarjetas afectadas
    · Reemplazado por opacity 0.85→1 (GPU compositor)

[4] Track Agenda: style.left + style.width → transform + scaleX + CSS var
    · Cambiar left/width = geometry recalc en cada frame
    · Ahora: fill usa scaleX(pct/100), icon usa --icon-pct CSS custom property
    · Ambas propiedades son compositor-only: cero layout recalc

[5] will-change:transform eliminado de TODAS las .card
    · will-change global en 8+ tarjetas = GPU layers permanentes = memory pressure
    · Ahora solo se activa en .card.card-animating durante animación activa

[6] void card.offsetWidth eliminado de _animateCardAppear
    · Forced synchronous layout — el reflow más costoso posible
    · Reemplazado por requestAnimationFrame (async, sin bloqueo)

[7] _applyDashMode cacheado
    · Antes se ejecutaba el DOM pass completo en cada render
    · Ahora compara modeKey y salta si nada cambió

[8] refresh() forEach loop eliminado
    · El loop que modificaba style.animation en todas las tarjetas
    · Causaba reflow cascada en cada pull-to-refresh
