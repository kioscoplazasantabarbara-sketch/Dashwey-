FASE 3 — FIXES APLICADOS
═══════════════════════════════════════════

Fix único: transition:left → transform:translateX en 3 familias de toggle

Los toggles de settings (cm-toggle, settings-toggle, sd-toggle) animaban la
propiedad CSS 'left', que fuerza layout recalculation + paint en cada frame.

ANTES: transition: left 0.22s var(--ease) → browser recalcula layout en cada frame
DESPUÉS: transition: transform 0.22s var(--ease) → compositor-only, 0ms layout

Cambios exactos:
  .cm-knob:                 left:3px/21px → transform:translateX(0)/translateX(18px)
  .cm-toggle.on .cm-knob:   left:21px     → transform:translateX(18px)
  .settings-toggle::after:  left:3px/21px → mismo patrón
  .settings-toggle.on::after: left:21px   → translateX(18px)
  .sd-toggle::after:        left:3px/21px → mismo patrón
  .sd-toggle.on::after:     left:21px     → translateX(18px)

will-change:transform añadido a los 3 elementos animados (scoped al pseudo-elemento,
no al toggle contenedor → GPU layer solo donde se necesita).
